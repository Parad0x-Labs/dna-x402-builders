# Security Policy

Do not open issues containing secrets, private keys, seed phrases, wallet recovery material, API tokens, or production URLs that are not already public.

Public Beta integrations must follow these rules:

- no backend custody
- no backend signing
- no hidden fees
- no auto-sweep
- client-side/user-owned signing for live flows
- receipt verification before unlocking paid results
- idempotent webhook handling

Report sensitive issues privately through Parad0x Labs' preferred security contact.
